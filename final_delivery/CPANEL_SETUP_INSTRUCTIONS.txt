                        CPANEL DATABASE SETUP GUIDE
═══════════════════════════════════════════════════════════════════════════

If you're getting HTTP 500 errors on referral_codes.php and stock_alerts.php, 
you need to create the missing database tables on cPanel.

QUICK FIX:
═════════

1. Go to cPanel → phpMyAdmin

2. Select your database (silentmu_silentdb or your database name)

3. Click "SQL" tab at the top

4. Copy ALL the code from: web_panel/CREATE_TABLES.sql

5. Paste it into the SQL window in phpMyAdmin

6. Click "Go" button

7. You should see: "2 queries executed successfully"

8. Refresh both pages:
   - referral_codes.php
   - stock_alerts.php

They should now work!

TROUBLESHOOTING:
═════════════════════════════════════════════════════════════════════════════

Q: "Access Denied" error?
A: You need admin access to phpMyAdmin. Contact your hosting provider.

Q: "Table already exists" error?
A: That's okay! It means the table is already there.
   The pages should work now.

Q: Still getting errors?
A: Check the error by:
   1. Enable error display in config/database.php (already enabled)
   2. Check your server error logs in cPanel

Q: Where is my database name?
A: Go to cPanel → MySQL Databases
   You should see it listed there (silentmu_silentdb)
